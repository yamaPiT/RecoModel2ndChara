| 測定項目 | 測定基準 | 記述例（適合例／非適合例） |
| ------ | ------ | ------ |
| 文書の読み手（属性や特性）を記述している | 対象文書の前提として、想定する読者の役割、職種、前提となる専門知識や業務スキルを具体的に定義している。 | ★新規追加【改行】【非適合例】本仕様書は、システム開発に関わる全担当者を対象とする。(読み手の役割や必要とする経験値・スキルに関する記述が曖昧で、不十分である)【改行】→【適合例】本仕様書は、基本設計工程を担当するアプリケーションエンジニアを対象とする。JavaおよびSpring Bootに関する基礎知識を有していることを前提とする。 |
| 文書の読み手の範囲（提出先,配布先,開示範囲）を記述している | 作成した文書や成果物について、社内外の提出先、関係者への配布先、および機密レベルに応じた情報の開示範囲を明記している。 | ★新規追加【改行】【非適合例】プロジェクト計画書において、各開発フェーズの具体的な成果物やスケジュールが記載されているが、その取り扱い方針が指定されていない。(成果物に対する提出先などの読み手の範囲が記載されていない)【改行】→【適合例】プロジェクト計画書において、各開発フェーズの成果物ごとに、提出先（顧客情報システム部）、配布先（プロジェクト内全メンバー）、および開示範囲（社外秘）を一覧として定義している。 |
| 文書の適用範囲を記述している | システムや文書が対象とする業務領域、システム化の範囲、および対象外となる境界線を明確に定義している。 | ★新規追加【改行】【非適合例】システム要件定義書において、顧客からヒアリングした業務要望をすべて要件として列挙している。(適用範囲が不明確なため、顧客のニーズに関係ない要件が多く含まれる原因となる)【改行】→【適合例】システム要件定義書において、国内向け販売管理業務における受注から出荷までの領域を適用範囲とし、顧客のニーズに基づく機能要件を明確に定義している。 |
| 文書の目的を記述している | 当該文書を作成する背景、および読者がその文書を用いて達成すべき目的を、冒頭などの明確な位置に明記している。 | ★新規追加【改行】【非適合例】ソフトウェア設計書の冒頭から、画面レイアウトやデータベースの仕様が羅列されている。(文書自体の目的が記載されていないため、目的と無関係な機能や仕様が多く記述される原因となる)【改行】→【適合例】本ソフトウェア設計書は、システムの目的と一致するアーキテクチャの設計方針を定義し、開発チーム全体で共有することを目的とする。 |
| 文書の目的を達成するために必要な項目を記述している | 要求定義を行う文書において、システムの方向性を決定づける開発の目的と対象スコープを明記している。 | ★新規追加【改行】【非適合例】要求仕様書の冒頭において、次期システムの機能一覧表のみが記載されている。(開発の目的が明示されていない)【改行】→【適合例】要求仕様書の「はじめに」の章に、業務効率化という開発の目的や、本プロジェクトにおける開発のスコープを記載している。 |
| 文書の目的を達成するために必要な項目を記述している | システムの品質基準を担保するため、要求を定義する文書内に機能要件だけでなく、非機能要求を網羅的に記載する専用の章や項目を設けている。 | ★新規追加【改行】【非適合例】要求仕様書において、ユーザーが利用する画面や帳票に関する機能要件のみが記載されている。(目的達成に必要な非機能要求に関する項目が欠落している)【改行】→【適合例】要求仕様書において、機能要件の章とは別に、非機能要求を記載する章を設けている。 |
| 文書の目的を達成するために必要な項目を記述している | 開発工程に応じた設計関連の文書において、そのフェーズで達成すべき技術的または業務的な設計の目的を明記している。 | ★新規追加【改行】【非適合例】基本設計書の冒頭から、システム構成図やインターフェースの定義が直ちに記載されている。(設計の目的が欠落している)【改行】→【適合例】基本設計書において、「本設計は、将来的なデータ量増加に耐えうる拡張性の高いデータベース構造を定義することを目的とする」と設計の目的を記載している。 |
| 文書の目的を達成するために必要な項目を記述している | 保守性や拡張性を担保するため、初期開発時だけでなく、将来的なシステムの改訂や機能追加を見据えた設計の目的を記述している。 | ★新規追加【改行】【非適合例】現在の業務要件を満たすための最短の実装方針のみを記載している。(将来の改訂開発に関する観点が欠落している)【改行】→【適合例】基本設計書において、将来の法改正に伴う帳票レイアウトの改訂開発を想定し、表示ロジックとデータ出力処理を分離することを設計の目的として記述している。 |
| 文書の目的を達成するために必要な項目を記述している | 会議の記録と合意形成という文書の目的を達成するため、フォーマット上に開催日時、参加者、議題、および決定事項を記載する定型の記入欄を設けている。 | ★新規追加【改行】【非適合例】議事録のテンプレートに、自由記述のメモ欄のみが存在する。(会議の目的を達成するために必要な項目を記載する欄がない)【改行】→【適合例】議事録のフォーマットには、ヘッダ部分に開催日、参加者、議題を書く欄があり、末尾に議決結果を書く欄を設けている。 |
