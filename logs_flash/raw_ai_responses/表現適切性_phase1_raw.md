| 測定項目 | 測定基準 | 記述例（適合例／非適合例） |
| ------ | ------ | ------ |
| 視点を統一している | ユーザインターフェースや業務フローを記述する際、操作主体であるユーザ側の挙動として一貫性のある記述を行っている。 | ★新規追加【改行】【非適合例】ユーザがボタンを押下すると、システムは画面を遷移させ、次にユーザは入力を行う。(同一文内で操作主体と処理主体の視点が混在している）【改行】→【適合例】ユーザは、スマートウォッチのボタンを押下して次画面へ遷移し、必要な情報を入力する。 |
| 視点を統一している | 機能仕様や内部処理を記述する際、処理主体であるシステム側の動作として一貫性のある記述を行っている。 | ★新規追加【改行】【非適合例】心拍数計測機能において、ユーザがセンサーに触れると数値が表示される。(設計書において動作主体がシステムではなくユーザ視点のみで記述されている）【改行】→【適合例】心拍数計測機能は、センサーからの入力を検知し、算出された心拍数を画面に表示する。 |
| 行為の主体を特定できる | システムの操作権限や実行手順において、その行為を行う責任者（ロール）やコンポーネント名を主語として明記している。 | 【非適合例】誰がシステムの操作権限を持っているか、文書内で特定できない。(操作権限を持つ主体が不明確である）【改行】→【適合例】要件仕様書において、管理責任者がユーザのアカウント削除権限を持つことを明記している。 |
| 箇条書きまたは図表を適切に使用している | 文書内の図表に対して、内容を識別するための固有の項番および名称（キャプション）を付与している。 | 【非適合例】図表にキャプションが付いておらず、本文で参照されていない。(図表の識別情報が欠落している）【改行】→【適合例】図 3-1 スマートウォッチ外観図のように、図表に項番およびキャプションを付けている。 |
| 箇条書きまたは図表を適切に使用している | 本文中で図表の内容を説明する際、図表番号を用いて対象となる図表を具体的に指し示している。 | ★新規追加【改行】【非適合例】下図の通り、データの流れを定義する。(複数の図がある場合にどの図を指しているか特定できない）【改行】→【適合例】図 4-2 データフロー図に示す通り、各処理間のデータ受け渡しを定義する。 |
| インデントまたは改行を効果的に使用している | 見出し、本文、リストなどの要素ごとに、統一されたルールに基づいて字下げや行間の空白を設定している。 | 【非適合例】見出しと本文でインデントや改行が不統一で、読みづらい。(視覚的な構造が整理されていない）【改行】→【適合例】見出しと本文の間でインデントや改行が適切に使われている。 |
| 読みやすいレイアウトやフォント種を使用している | 全ページのヘッダーまたはフッターの定位置に、参照中の文書名を一貫して表示している。 | 【非適合例】ヘッダーに文書名が記載されていない。(印刷時などにどの文書の一部か判別できない）【改行】→【適合例】全ページのヘッダーに、文書名「基本設計書」を記載している。 |
| 読みやすいレイアウトやフォント種を使用している | 文書の全ページに対して、一連の番号（ノンブル）を、ページ内の統一された位置に付与している。 | 【非適合例】ページ番号がない。(会議等で特定の箇所を指し示すことが困難である）【改行】→【適合例】本文の各ページ下部中央に、ノンブル（ページ番号）を記載している。 |
