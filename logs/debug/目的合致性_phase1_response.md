| 測定項目 | 測定基準 | 記述例（適合例／非適合例） |
| ------ | ------ | ------ |
| 明示した目的を達成するために必要な内容を記述している | 各記述単位（章・節・項・段落・図・表等）の内容を、その単位の目的に合致させている。 | ★新規追加【改行】【非適合例】「第2章 用語の定義」の節において、用語の説明ではなく、システムの機能要件を記述している。 (記述内容が節の目的に合致していない)【改行】→【適合例】「第2章 用語の定義」の節において、本ドキュメントで使用する独自の専門用語とその定義を記述している。 |
| 明示した目的を達成するために必要な内容を記述している | 文書の対象範囲を遵守し、後続工程で決定すべき事項を含めずに記述している。 | ★新規追加【改行】【非適合例】要求仕様書において、特定のプログラミング言語における内部関数の引数構成や命名規則を詳細に記述している。 (次工程以降で決定すべき実装詳細が記載されている)【改行】→【適合例】要求仕様書において、システムが満たすべき機能要件と、外部システムとのインターフェース条件を記述している。 |
| 明示した目的を達成するために必要な内容を記述している | 目的達成のために不可欠な情報、条件、および相手に求める行動を具体的に記述している。 | ★新規追加【改行】【非適合例】納期延長を依頼するE-mailにおいて、開発が滞っている状況報告のみを行い、希望の納期や延長の依頼を記述していない。 (相手に求める具体的なアクションが明示されていない)【改行】→【適合例】納期延長を依頼するE-mailにおいて、遅延の理由とともに「納期を〇月〇日まで延長いただきたい」という具体的な要求を記述している。 |
| 明示した目的を達成するために必要な内容を記述している | 記述内容の根拠や前提となる設計方針、および作成の意図を記述している。 | ★新規追加【改行】【非適合例】基本設計書において、具体的な画面レイアウトのみを記載し、その構成を採用した根拠を記述していない。 (設計の背景にある方針や意図が不明である)【改行】→【適合例】基本設計書において、「操作ミスを最小化する」という設計方針と、それに基づき実行ボタンを孤立させた意図を記述している。 |
