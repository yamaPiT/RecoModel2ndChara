| 測定項目 | 測定基準 | 記述例（適合例／非適合例） |
| ------ | ------ | ------ |
| 複数の解釈が成り立たない | 接続助詞「～し」や「～して」等の、並列、追加、原因、帰結などの複数の意味を持つ表現を避け、前後関係を一意に特定できる記述を行っている。 | 【非適合例】データを保存し、メールを送信する。（「保存した後に送信」か「保存と並行して送信」かが不明）【改行】→【適合例】データを保存した後に、メールを送信する。 |
| 複数の解釈が成り立たない | 「全ての」「必ずしも」等の修飾語と否定形を組み合わせる際、対象の全範囲を否定するのか一部を否定するのかが、文理上客観的に判別できる表現を選択している。 | 【非適合例】[例文2]全ての処理が成功しない場合は手順Cに進む。（「一部が失敗したとき」か「全てが失敗したとき」かが不明）【改行】→【適合例】[例文2a]1つ以上の処理が失敗した場合は、手順Cに進む。 |
| 複数の解釈が成り立たない | 図表、フローチャート、構成図等への言及において、それが既にある事実の「提示」なのか、設計上の構造を「定義・決定」するのか、その意図が明確になる動詞を選択している。 | 【非適合例】[例文1]プログラムの構造を図1に示す。（図1が構造の解説図なのか、構造を決定する設計図なのかが曖昧）【改行】→【適合例】[例文1b]図1の構成により、プログラムの構造を定義する。 |
| 複数の解釈が成り立たない | 修飾語と被修飾語を可能な限り隣接させ、かつ長い修飾語を文頭側に配置することで、係り受けの関係が単一の意味にのみ解釈できる語順を採用している。 | ★新規追加【改行】【非適合例】迅速に設計した仕様書を確認する。（「迅速に」が「設計」にかかるのか「確認」にかかるのかが不明確）【改行】→【適合例】設計が完了した仕様書の内容を、迅速に確認する。 |
| 複数の解釈が成り立たない | 文の「主語と述語」を可能な限り近接させ、一文の中に介在する要素を最小化することで、動作の主体と内容が即座に理解できる構造を維持している。 | ★新規追加【改行】【非適合例】本機能は、前回のリリースで実装された共通モジュールを呼び出すことで、ユーザーから入力されたデータのバリデーションを行う。（主語と述語が離れすぎている）【改行】→【適合例】本機能は、ユーザーの入力データをバリデーションする。その際、前回実装された共通モジュールを呼び出す。 |
| １つの文内で１つの事柄のみを記述している | 句点（。）で区切られた一文の中で、表現する要件、動作、または定義を1つに限定し、論理的な意味単位（一文一義）を遵守して記述している。 | 【非適合例】ユーザーIDを入力してログインボタンを押下すると、認証サーバーに問い合わせが行われ、結果が画面に表示される。（入力、処理、表示が1文に混在）【改行】→【適合例】ユーザーIDを入力してログインボタンを押下する。システムは認証サーバーにIDの有効性を問い合わせ、その結果を画面に表示する。 |
| 同じ内容のものを違う名前で定義しない。 | 文書全体または用語定義集等の外部仕様に基づき、同一の概念、データ項目、機能名に対して、唯一の名称を継続して使用している。 | 【非適合例】同一の概念に対して異なる名称が混在している。（「顧客ID」と「ユーザーコード」が同じ対象を指している状態）【改行】→【適合例】同一の対象に対し、文書全体で「顧客ID」という用語を統一して使用している。 |
| 指し示すものが明確である | 「前述」「上記」「前回」などの相対的な位置や順序を示す表現を排除し、参照先の章番号、項目名、または具体的な処理IDを明記して対象を特定している。 | 【非適合例】［例文1］前回の処理と同様とする。（どの工程やステップを指すかが一意に特定できない）【改行】→【適合例】[例文1a]承認ボタンが押下されたときと同一の処理を実行する。 |
| 指し示すものが明確である | 「来月」「現在」等の記述時点を基準とした直示表現を用いず、西暦や具体的な日付などの絶対的な指標を用いて時間的対象を確定している。 | 【非適合例】[例文2]昨年12月のリリース記録を参照。（「昨年」が指す年が、文書の閲覧時期によって変化する）【改行】→【適合例】[例文2a]2023年12月のリリース記録を参照する。 |
| 指し示すものが明確である | 指示代名詞（これ、それ等）の使用を最小限に留め、使用する場合はその指示対象が直前のどの語句を指しているかが客観的に一意に特定できる構文を選択している。 | ★新規追加【改行】【非適合例】サーバーの負荷が増大し、レスポンスが低下した。これがシステムの問題である。（「これ」が負荷増大かレスポンス低下かが不明確）【改行】→【適合例】サーバーの負荷が増大し、レスポンスが低下した。このレスポンスの低下が、本システムにおける改善課題である。 |
| 表に空欄を残さない | 表形式の記載において、値が存在しない、未定、または該当しないセルに対し、特定の記号（「-」「N/A」「なし」等）を記入し、意図的な無記入であることを明示している。 | 【非適合例】表内の空欄が、該当なし、未定（TBD）、記入漏れのいずれを意味するかが判別できない。（情報の欠落と意図的な空欄が区別不能）【改行】→【適合例】設定値が存在しないセルには、一律で「-」を記載する。 |
| 状態とイベント（事象）を区別している | システムの継続的な「状態（State）」と、変化の契機となる「事象（Event）」を、助詞「～のとき」と「～したとき」や、文末の表現を使い分けて厳格に区別している。 | 【非適合例】[例文1]電源ONのときに、処理Aを実行する。（「ONの状態」か「ONに切り替わった瞬間」かが曖昧）【改行】→【適合例】［例文１a］電源の状態がOFFからONに変化したときに、処理Aを実行する。 |
