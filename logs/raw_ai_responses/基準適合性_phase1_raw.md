| 測定項目 | 測定基準 | 記述例（適合例／非適合例） |
| ------ | ------ | ------ |
| 職務分掌に則った権限委譲などのビジネスルールに従っている | 対象業務のルール（職務分掌や権限委譲など）に基づき、各機能に対する利用者のアクセス権限や操作可能範囲の制御仕様を明記している。 | 【非適合例】アクセス権限が設定されていないため、発注処理が無制限に行える。(ビジネスルールに基づく権限制御が未定義)【改行】→【適合例】所属部署以外から発注処理を行えないように、アクセス権限を設定している。 |
| 国内だけでなく、グローバルな環境にも適合している | システムを導入または提供する対象国および地域の法令、各種規制、業界標準などの法的要件を特定し、それらを満たす仕様を明記している。 | ★新規追加【改行】【非適合例】エンジン制御システム開発において、自動車排出ガスが日本国内の基準値のみを満たす仕様となっている。(グローバルな環境に適合せず、自国の規制しか考慮されていない)【改行】→【適合例】エンジン制御システム開発で、自動車排出ガスが、仕向地の規制値を満たしている。 |
| 国内だけでなく、グローバルな環境にも適合している | グローバルに展開するサービスにおいて、現地の実情（連絡先、言語、タイムゾーン等）を考慮し、対象国の利用者が実際に利用可能な仕様として定義している。 | ★新規追加【改行】【非適合例】グローバルに利用可能なサービスに関して、問い合わせ先に無料通話番号が載っているが、異国からは無料で電話がかけられない。(自国のみにしか適用できない記述をしている)【改行】→【適合例】グローバルに利用可能なサービスに関して、対象となる各国から無料で電話がかけられる現地の通話番号を記載している。 |
| 技術領域の基準または標準に準拠している | 採用する技術、アルゴリズム、または外部ライブラリが、第三者の知的財産権を侵害していないことを調査し、合法性を確認した根拠を記述している。 | ★新規追加【改行】【非適合例】実装予定のアルゴリズムについて先行技術調査が実施されておらず、特許を侵害している可能性がある。(特許侵害リスクの確認が不明確である)【改行】→【適合例】実装予定のアルゴリズムについて先行特許調査を実施し、第三者の特許を侵害していないことを確認している。 |
| 技術領域の基準または標準に準拠している | システムが取り扱う機密データについて、法令や組織のセキュリティ基準に基づく管理対象として特定し、その保護仕様を定義している。 | ★新規追加【改行】【非適合例】個人情報データベースに対するアクセス制御や暗号化の仕様が定義されていない。(法令や組織の規定に基づく管理仕様が不明確である)【改行】→【適合例】国や会社などの定めに基づいて、個人情報データベースを管理している。 |
| 文書標準に準拠している | 組織やプロジェクトが指定する文書のひな型を採用し、規定された章立てや入力フォーマットに従って各項目を記述している。 | ★新規追加【改行】【非適合例】基本設計書において、プロジェクト標準の章立てを無視し、独自の構成で機能要件を記述している。(テンプレートに従わずに記述されている)【改行】→【適合例】プロジェクト標準の基本設計書テンプレートに従って記述している。 |
| 文書標準に準拠している | 外部資料や他システムの仕様書から情報を参照する場合、対象となる文書名、バージョン、または該当箇所を情報源として明記している。 | ★新規追加【改行】【非適合例】外部連携APIの仕様について、参照した公式ドキュメント名やバージョンが記載されていない。(情報源となる引用元が明示されていない)【改行】→【適合例】外部連携APIの仕様について、提供元が公開している「API連携仕様書（v2.0）」を引用元を明示している。 |
| 事実と意見を書き分けている | 客観的に確認可能な事象（技術的な制約、テスト結果、絶対条件など）と、主観的な判断（要求仕様案、設計結果、対応方針など）を、文脈上明確に分離して記述している。 | ★新規追加【改行】【非適合例】負荷テストで処理遅延が発生しているため、本番環境のサーバースペックをスケールアップすべきである。(事実と意見が混同されて記述されている)【改行】→【適合例】負荷テストの結果、処理遅延が発生した（事実：テスト結果）。これを解消するため、本番環境のサーバースペックのスケールアップを提案する（意見：テスト結果に基づくデバッグ方針）。 |
