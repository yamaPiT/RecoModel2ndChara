| 測定項目 | 測定基準 | 記述例（適合例／非適合例） |
| ------ | ------ | ------ |
| 動作を特定できる | 抽象的な動作表現を避け、設定すべき具体的な値や実行する処理名を用いて記述している。 | 【非適合例】変数Aをリセットする。 (リセットの具体的な動作内容が不明であるため)【改行】→【適合例】変数Aに初期値の1を代入する。 |
| 状態を特定できる | 状態や性質を示す用語に対して、具体的な判定条件や定量的な閾値を明記して定義している。 | ★新規追加【改行】【非適合例】センサAの読み取り値が異常な場合に、過負荷状態に遷移する。 (「異常」という状態の明確な定義がないため)【改行】→【適合例】センサAの読み取り値が、2回連続して99を超えた場合に過負荷状態に遷移する。 |
| 対象を特定できる | 動作や説明の対象となる構成要素を省略せず、具体的な名称を用いて記述している。 | 【非適合例】構造を説明する。 (何の構造であるかが不明であるため)【改行】→【適合例】次の章で、流量センサのセンシング構造を説明する。 |
| 対象を特定できる | 物理的な配置や論理的な位置を示す場合、図番や座標を用いて一意に特定可能な状態で記述している。 | 【非適合例】スイッチをパネルの上部に取り付ける。 (「上部」だけでは正確な位置を特定できないため)【改行】→【適合例】スイッチを図2の位置に取り付ける。 |
| 対象を特定できる | 動作を表す動詞を使用する際、操作の対象となる目的語を明示して記述している。 | ★新規追加【改行】【非適合例】入力データを受信した後、分割する。 (分割の対象物を明示していないため)【改行】→【適合例】入力データを受信した後、入力データをヘッダとペイロードに分割する。 |
| 違う内容のものを同じ名前で定義しない | 文書内で定義される用語や識別子において、異なる概念や対象に対して同一の名称を使用せず、一意な名称を割り当てている。 | ★新規追加【改行】【非適合例】プロセスの状態を示すステータスと、通信結果を示すステータスを定義する。 (異なる概念に対して「ステータス」という同じ名前を使用しているため)【改行】→【適合例】プロセスの状態を示すプロセス状態区分と、通信結果を示す通信結果コードを定義する。 |
| 意味を特定しやすい助詞を使用している | 動作や範囲の起点を表す場合、起点を明示する専用の助詞を用いて記述している。 | 【非適合例】このテスト項目より、テストを始める。 (起点に対して比較を意味する助詞を使用しているため)【改行】→【適合例】このテスト項目から、テストを始める。 |
| 意味を特定しやすい助詞を使用している | 対象間の比較を表す場合、比較の基準となる対象を明示する助詞を用いて記述している。 | ★新規追加【改行】【非適合例】LED2は、LED1に対して高輝度である。 (比較を明示する標準的な助詞を使用していないため)【改行】→【適合例】LED2は、LED1よりも高輝度である。 |
| 意味を特定しやすい助詞を使用している | 動作の到達点や結果の帰着点を示す場合、終点を示す専用の助詞を用いて記述している。 | 【非適合例】設計のレビュー対応が完了したら、実装工程へ進む。 (終点に対して方向を意味する助詞を使用しているため)【改行】→【適合例】設計のレビュー対応が完了したら、実装工程に進む。 |
