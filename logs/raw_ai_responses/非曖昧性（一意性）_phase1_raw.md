| 測定項目 | 測定基準 | 記述例（適合例／非適合例） |
| ------ | ------ | ------ |
| 複数の解釈が成り立たない | 接続助詞「～し」や「～して」を使用せず、前後の節の関係が「並列・追加」なのか「原因・帰結」なのかを一意に特定できる表現を用いている。 | 【非適合例】データを保存し、メールを送信する。（中止法によって、保存後に送信するのか並行して送信するのかが不明）【改行】→【適合例】データを保存した後に、メールを送信する。 |
| 複数の解釈が成り立たない | 否定語を用いる際、対象のすべてを否定する「全体否定」か、一部を否定する「部分否定」かが客観的に判別できる表現を選択している。 | 【非適合例】[例文2]全ての処理が成功しない場合は手順Cに進む。（全体否定か部分否定かがわからない）【改行】→【適合例】[例文2a]全ての処理が失敗した場合は手順Cに進む。 |
| 複数の解釈が成り立たない | 文中の図表への言及が、既にある事実の「記載・提示」を指すのか、その図表を用いて構造を「定義・決定」するのかを明確に区別して記述している。 | 【非適合例】[例文1]プログラムの構造を図1に示す。（すでに決まっている構造を提示しているのか、図1によって構造を決定しているのかが曖昧）【改行】→【適合例】[例文1b]図1によりプログラムの構造を定める。 |
| 複数の解釈が成り立たない | 被修飾語と修飾語の間に他の語句を介在させず、係り受けの関係が一意に特定できる語順を選択している。 | ★新規追加【改行】【非適合例】迅速に設計した仕様書を確認する。（「迅速に」が「設計」にかかるのか「確認」にかかるのかが不明確）【改行】→【適合例】設計した仕様書の内容を、迅速に確認する。 |
| 複数の解釈が成り立たない | 文の「主語と述語」および「修飾語と被修飾語」を可能な限り近接させ、構造の複雑化による解釈のブレを排除している。 | ★新規追加【改行】【非適合例】本機能は、前回のリリースで実装された共通モジュールを呼び出すことで、ユーザーから入力されたデータのバリデーションを行う。（主語と述語が離れすぎており、動作の主体と内容が即座に理解できない）【改行】→【適合例】本機能は、ユーザーの入力データをバリデーションする。その際、前回実装された共通モジュールを呼び出す。 |
| １つの文内で１つの事柄のみを記述している | 1つの文（句点から句点まで）には独立した意味を持つ事柄を1つに絞って記述し、複数の要件や動作を混在させない。 | 【非適合例】1つの文の中に複数の事柄が混在している。（要件定義などで、入力・処理・出力が1つの長文にまとめられている状態）【改行】→【適合例】1つの文には、1つの事柄だけが記述されている。 |
| 同じ内容のものを違う名前で定義しない。 | 文書全体、またはプロジェクトの用語定義集に基づき、同一の概念、エンティティ、機能に対して唯一の名称を継続して使用している。 | 【非適合例】同じ概念に対して異なる名称が使われている。（「顧客ID」と「ユーザーコード」が同じものを指しているが混在している状態）【改行】→【適合例】同じ内容について、文書全体で統一された用語が使用されている。 |
| 指し示すものが明確である | 「前回」「前述」などの相対的な順序を示す表現を避け、対象となる処理、節、項目を具体名や管理番号で特定している。 | 【非適合例】［例文1］前回の処理と同様とする。（「前回」がどの工程やステップを指すかを明示していない）【改行】→【適合例】[例文1a]承認ボタンが押下されたときと同じ処理をする。 |
| 指し示すものが明確である | 「昨年」「来月」などの記述時点に依存する直示表現を使用せず、西暦などの絶対的な時間指標を用いて対象を特定している。 | 【非適合例】[例文2]昨年12月のリリース記録を参照。（直示表現「昨年」が、参照する時点によって変わり曖昧になる）【改行】→【適合例】[例文2a]2023年12月のリリース記録を参照。 |
| 指し示すものが明確である | 指示語（これ、それ、あれ、どれ等）を使用する場合、その指示対象が直前のどの語句を指しているかが客観的に一意に解釈できる。 | ★新規追加【改行】【非適合例】サーバーの負荷が増大し、レスポンスが低下した。これがシステムの問題である。（「これ」が負荷増大を指すのかレスポンス低下を指すのかが不明確）【改行】→【適合例】サーバーの負荷が増大し、レスポンスが低下した。このレスポンスの低下がシステムの問題である。 |
| 表に空欄を残さない | 表形式のデータにおいて、値が存在しない、または未定であるセルには、特定の記号（斜線や「N/A」等）を記載し、意図的な空欄であることを明示している。 | 【非適合例】表内の空欄が、該当なし、TBD（未定）、記入忘れのいずれの意味にも解釈できる。（情報の欠落と意図的な空欄が区別できない）【改行】→【適合例】書き込む値がない表のセルには、斜線を引く。 |
| 状態とイベント（事象）を区別している | システムの「継続的な状態」と、変化の瞬間である「イベント（トリガー）」を、文末表現や助詞を用いて明確に書き分けている。 | 【非適合例】[例文1]電源ONのときに、~を実行する。（「ONの状態のとき」か、「ONに変わったとき」というイベントかが曖昧）【改行】→【適合例】［例文１a］電源がOFFからONに変わったときに、~を実行する。 |
