| 測定項目 | 測定基準 | 記述例（適合例／非適合例） |
| ------ | ------ | ------ |
| 動作を特定できる | システムが実行する操作を記述する際、システム振る舞い定義やデータ仕様に基づき、代入する具体的な数値、呼び出す関数名、または状態変化の完了条件を明記している。 | 【非適合例】変数Aをリセットする。（「リセット」が初期値の代入かメモリの解放か特定できないため。）【改行】→【適合例】変数Aに初期値の1を代入する。 |
| 状態を特定できる | 機器やソフトウェアの状態遷移条件を記述する際、要件定義書に定められた定量的な閾値、継続時間、または発生回数を明記して判定基準を定義している。 | ★新規追加【改行】【非適合例】センサAの読み取り値が異常な場合に、過負荷状態に遷移する。（「異常」とみなすための具体的な数値や継続時間の定義が欠落しているため。）【改行】→【適合例】センサAの読み取り値が2回連続して99を超えた場合に、過負荷状態に遷移する。 |
| 対象を特定できる | システム構成要素の動作や仕様を説明する際、指示代名詞や文脈依存の省略を排除し、システム構成図や用語集に定義された正式名称を用いて対象を明記している。 | 【非適合例】構造を説明する。（説明の対象となる具体的なモジュール名や機能名が省略されているため。）【改行】→【適合例】次の章で、流量センサのセンシング構造を説明する。 |
| 対象を特定できる | ハードウェアの物理的な配置やUIの論理的な配置を指示する際、曖昧な方向詞を避け、設計図面の図番、絶対座標、または相対的な寸法を用いて一意に特定できる状態で記述している。 | 【非適合例】スイッチをパネルの上部に取り付ける。（「上部」が指し示す範囲が広く、実装者によって取り付け位置の解釈が分かれるため。）【改行】→【適合例】スイッチを図2の位置に取り付ける。 |
| 対象を特定できる | データの加工や制御を実行する際、処理の対象となる入力データや出力先を変数名やデータ構造の名称を用いて目的語として明示している。 | ★新規追加【改行】【非適合例】入力データを受信した後、分割する。（「分割する」という処理の対象データと分割後の構成要素が明示されていないため。）【改行】→【適合例】入力データを受信した後、入力データをヘッダとペイロードに分割する。 |
| 違う内容のものを同じ名前で定義しない | データモデルやアーキテクチャ設計において、異なる役割を持つ概念や変数に対して、データディクショナリに基づきそれぞれ独立した固有の名称を割り当てている。 | ★新規追加【改行】【非適合例】プロセスの状態を示すステータスと、通信結果を示すステータスを定義する。（異なる属性情報に対して「ステータス」という同一の抽象名詞を使い回しているため。）【改行】→【適合例】プロセスの状態を示すプロセス状態区分と、通信結果を示す通信結果コードを定義する。 |
| 意味を特定しやすい助詞を使用している | 処理の開始位置や有効範囲の起点を指定する際、比較を表す「より」を排除し、起点を一意に定める助詞「から」を用いて記述している。 | 【非適合例】このテスト項目より、テストを始める。（空間的・時間的な起点に対して、比較の助詞「より」を誤用しているため。）【改行】→【適合例】このテスト項目から、テストを始める。 |
| 意味を特定しやすい助詞を使用している | 複数のコンポーネント間で性能や仕様を比較する際、対象関係を表す「に対して」を排除し、比較の基準を明確に示す助詞「よりも」を用いて記述している。 | ★新規追加【改行】【非適合例】LED2は、LED1に対して高輝度である。（対比や動作の対象を示す「に対して」を使用しており、比較の基準が直感的に伝わらないため。）【改行】→【適合例】LED2は、LED1よりも高輝度である。 |
| 意味を特定しやすい助詞を使用している | 状態遷移の帰着点やデータの保存先を指定する際、方向を表す「へ」を排除し、到達点や帰着点を明確に示す助詞「に」を用いて記述している。 | 【非適合例】設計のレビュー対応が完了したら、実装工程へ進む。（状態の確定的な到達点に対して、単なる方向を示す助詞「へ」を誤用しているため。）【改行】→【適合例】設計のレビュー対応が完了したら、実装工程に進む。 |
