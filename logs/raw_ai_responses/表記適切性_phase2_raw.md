| 測定項目 | 測定基準 | 記述例（適合例／非適合例） |
| ------ | ------ | ------ |
| 文体を統一している | プロジェクトの文書執筆規約に基づき、文書全体の文末表現を常体（だ・である調）または敬体（です・ます調）のいずれか一方に統一している。 | ★新規追加【改行】【非適合例】本システムは、ユーザ情報を管理する。また、パスワードの有効期限は90日としています。（非適合の理由：同一段落内で常体と敬体が混在している。）【改行】→【適合例】本システムは、ユーザ情報を管理する。また、パスワードの有効期限は90日としている。 |
| 用語および表記法を統一している | プロジェクトの用語定義集や表記ガイドラインに準拠し、同一文書内で同義語の表記揺れ（外来語の長音符、送り仮名、アルファベットの大文字小文字など）を発生させず、単一の表記を採用している。 | ★新規追加【改行】【非適合例】データベースサーバの構築後、Webサーバーとの通信確認を行う。（非適合の理由：同一文内で外来語の長音符の有無が混在している。）【改行】→【適合例】データベースサーバの構築後、Webサーバとの通信確認を行う。 |
| 用語を使い分けている | 文書の想定読者（顧客、外部ベンダ、運用担当者など）のITリテラシやドメイン知識を定義したうえで、組織内でのみ通用する隠語を排除し、読者層が正確に理解できる標準的な用語へ置換している。 | ★新規追加【改行】【非適合例】夜間バッチがコケた場合、シス管にアラートを飛ばす。（非適合の理由：顧客や外部ベンダが閲覧する文書において、組織内の隠語や略語を使用している。）【改行】→【適合例】夜間バッチ処理が異常終了した場合、システム管理者にアラート通知を行う。 |
| 用語（専門用語，略語など）および記号を定義している（「再利用性」に関連） | 文書内で使用する専門用語や略語について、初出時にフルスペルと意味を併記するか、プロジェクト共通の用語定義集への参照を明記し、読者による解釈のブレを防止している。 | ★新規追加【改行】【非適合例】本機能は、BFF層でデータの集約処理を行う。（非適合の理由：専門的な略語が初出時にフルスペルや意味の定義なしに使用されている。）【改行】→【適合例】本機能は、BFF（Backends For Frontends）層でデータの集約処理を行う。 |
| 用語（専門用語，略語など）および記号を定義している（「再利用性」に関連） | 閾値や制約事項を示す条件式について、曖昧な記号（波線など）を排除し、数値の境界条件（以上、以下、未満、より大きいなど）を解釈の余地がない厳密な言語表現で定義している。 | ★新規追加【改行】【非適合例】パスワードの文字数は8文字〜16文字とする。（非適合の理由：波線記号を使用しており、境界値を含むかどうかが曖昧である。）【改行】→【適合例】パスワードの文字数は8文字以上16文字以下とする。 |
| 用字用語が適切である | 日本語で表現可能な一般的な動作や状態について、読解を妨げるIT業界特有のカタカナ語を排除し、標準的な和語や漢語に置き換えて記述している。 | ★新規追加【改行】【非適合例】要件定義でアグリーした仕様をベースに設計をフィックスする。（非適合の理由：日本語で表現できる動作に対して、不必要なカタカナ語を使用している。）【改行】→【適合例】要件定義で合意した仕様に基づき、設計を確定する。 |
| 用字用語が適切である | 接続詞、副詞、補助動詞などの表記において、JTF日本語標準スタイルガイドや「公用文作成の考え方」に準拠し、漢字ではなくひらがなを使用している。 | ★新規追加【改行】【非適合例】サーバを停止する際は、予めユーザへ告知する。（非適合の理由：スタイルガイドでひらがな表記が推奨されている副詞を漢字で表記している。）【改行】→【適合例】サーバを停止する際は、あらかじめユーザへ告知する。 |
| 用字用語が適切である | 一般的な用字用語集（記者ハンドブックなど）やプロジェクト固有の執筆ルールに基づき、同音異義語や類義語の中から、文脈の意図を正確に伝達する漢字や用語を選択している。 | ★新規追加【改行】【非適合例】システム障害によって発生した顧客の損害を保障する。（非適合の理由：損害を金銭で埋め合わせる意味である「補償」ではなく、権利を守る意味の「保障」を誤用している。）【改行】→【適合例】システム障害によって発生した顧客の損害を補償する。 |
| 記号類・括弧類を適切に使用している | プロジェクトのドキュメント規約に基づき、全角の括弧類と半角の引用符を、画面名、ボタン名、ソースコードの文字列リテラルなどの対象要素ごとに厳密に使い分けている。 | ★新規追加【改行】【非適合例】設定ファイルのキー名として“timeout"を指定する。（非適合の理由：文字列リテラルを囲む記号に、全角のカーリークォートと半角のストレートクォートが混在している。）【改行】→【適合例】設定ファイルのキー名として"timeout"を指定する。 |
